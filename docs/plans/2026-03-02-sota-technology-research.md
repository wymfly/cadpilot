# 端到端 3D 打印管线：2025/2026 最新 SOTA 技术预研与选型

为了确保我们的 9 个核心节点能够使用到当前（2025-2026）最好、最成熟的技术，我对各大开源社区（HuggingFace, GitHub 等）和相关底层算法库进行了深度调研。以下是针对关键技术卡点的最新 SOTA 选型报告。

---

## 1. 艺术大脑生成节点 (`generate_raw_mesh`)：Text/Image to 3D

目前 Hugging Face 上的 3D 生成已经全面从缓慢的优化方法（如早期的 DreamFusion）转向了**前馈扩散模型 (Feed-forward Diffusion)**，这使得在我们的流水线中实现“秒级生成”成为了可能。

**最新 SOTA 推荐 (2025年重点关注)：**

1.  **Hunyuan3D-2.0 (腾讯)**
    *   **地位**: 2024年底/2025年初发布，目前最强大的统一框架之一。
    *   **优势**: 支持图生3D和文生3D。相较于初代，它生成的网格在几何一致性和拓扑结构上有了质的飞跃。它是目前生成高质量可打印“毛坯”的首选。
2.  **Stable Fast 3D / SPAR3D (Stability AI)**
    *   **地位**: 追求极致速度的王者。
    *   **优势**: 从单张图片生成带贴图的 3D Mesh 仅需 **不到 0.5 秒**。如果我们的 Agent 管线追求交互式极速预览，这是最好的选择。
3.  **TRELLIS (NVIDIA / Stanford)**
    *   **优势**: 采用结构化潜空间 (Structured Latent) 方法。它在处理复杂形状时生成的多边形拓扑结构更接近专业人工建模，这对于我们后期的清洗修复极度有利。
4.  **MeshAnything V2 (后处理增强)**
    *   **应用场景**: 这个模型非常特别。它不是直接无中生有，而是接收 Hunyuan 或 TRELLIS 产生的“脏/密集网格”，将其转化为**拓扑干净、低多边形（<1600面）、完全闭合的网格**。
    *   **结论**: 强烈建议在 `generate_raw_mesh` 节点内部，采用 **[Hunyuan3D-2.0] + [MeshAnything V2]** 的级联模式，这能极大地减轻后续水密化修复（Node 4）的压力。

## 2. 几何内核与基础操作 (`generate_cadquery` 及底层支撑)

我们目前的基石是 `CadQuery`（底层为 OpenCASCADE / OCCT）。然而，在探索替代方案时，我们发现 Python 生态发生了巨大的变化。

**最佳底层内核选择：MeshLib (极其关键的发现)**

*   **传统 OCCT 的困境**: CadQuery 极其适合做参数化（画孔、拉伸），但在处理大量面或非标准拓扑时性能极差且容易崩溃。
*   **MeshLib (由 C++ 编写，提供一流的 Python 绑定)**:
    *   **为什么重要**: 它是目前 Python 生态中处理网格最现代化、速度最快的库（双重许可，开源 LGPL）。
    *   **核心能力**: 它可以在几秒内处理数百万个三角形的复杂布尔运算、自动修复非流形几何（Auto-repair）、以及进行极速的网格偏置 (Offset)。
    *   **结论**: 我们的架构应保持双轨制。**参数化建模依然用 `CadQuery`，但所有涉及“修复、检测、切片预处理”的网格操作，应全面转向 `MeshLib`**。

---

## 3. 混合布尔拼装 (`boolean_assemble`)

这是连接“艺术”与“工业”的桥梁。传统的库（如 Trimesh 封装的 Blender 或 OpenSCAD）在处理高精度重合面时几乎必死。

**SOTA 选型：Manifold3D vs MeshLib**

1.  **Manifold3D (目前已知最稳)**:
    *   **背景**: 前 Google 工程师使用 C++ 编写的无边界流形内核。
    *   **优势**: 速度极快，且在算法上**保证**布尔运算的结果一定是流形（Manifold，即完全水密、无飞面）。这是目前 Python 做 3D 混合装配唯一的“真神”。
2.  **MeshLib**:
    *   也具备极强的布尔运算能力，并且使用了 "Simulation of Simplicity (SoS)" 算法来处理退化/共面情况。

**结论**: `boolean_assemble` 节点首选引入 **`manifold3d`** 库，备选用 **`meshlib`**。坚决摒弃 `scipy.spatial` 或基础 `trimesh` 的布尔功能。

---

## 4. 树状支撑生成 (`generate_supports`)

切片软件里的树状支撑（Tree Supports）对于有机模型打印至关重要。如果我们想在管线内部（而不是交由 Cura）实现它，最新的算法路径如下：

**Python 中的实现路线：空间殖民算法 (Space Colonization Algorithm) + 极速射线检测**

1.  **悬垂检测**: 利用 `trimesh` 计算面法线与 Z 轴的夹角，标记需要支撑的悬垂区域（如 > 45°）。
2.  **采样与避障**: 在悬垂面上采样（生成 Attractor Points）。关键在于引入 `trimesh.ray.ray_pyembree` (极大地加速光线追踪)。
3.  **树干生长**: 使用类似于生成神经元或树枝的“空间殖民算法”。从台面开始生长节点，节点会自动寻找距离最近的悬垂采样点并向其靠拢，遇到模型本体（通过光线追踪预测碰撞）则绕行。
4.  **网格化**: 最终将生成的线段用 `trimesh.creation.cylinder` 转为圆柱体网格。

**结论**: 这在纯 Python 中**完全可行且有开源实现基础**。相比于让 Cura 生成黑盒支撑，我们在 Phase 2 可以实现自定义的 Python 支撑节点，从而能在网页端实时预览支撑结构。

## 5. 模型清洗与修复 (`mesh_healer`) - 重大突破策略

之前我们认为这是“生死劫”，主要卡在难以在 Python 中使用 OpenVDB。基于最新的科研，我们找到了破局之道。

**SOTA 破局策略：MeshLib 一把梭**

*   放弃配置痛苦的 `pyopenvdb`。
*   全面引入 **`meshlib`**。MeshLib 原生提供了基于体素（Voxel）和拓扑分析的高级自动修复功能。它能做到：闭合孔洞、统一法线、并且在体素化重采样时，比 `trimesh` 更好地保留原始特征。
*   如果遇到极度残缺的视觉模型，结合第一点提到的 **`MeshAnything V2`**（AI 后处理模型），将彻底解决这个困扰 3D 打印界的难题。

---

## 总结：更新后的技术栈武器库

基于 2026 年的 SOTA 调研，我们的 Agent 将手握以下最顶级的武器库：

*   **机械生成**: LLM + JSON AST + `CadQuery`
*   **艺术生成**: `Hunyuan3D-2.0` (生坯) + `MeshAnything V2` (AI 拓扑修复)
*   **网格清洗与几何处理**: **`MeshLib`** (取代大部分 Trimesh 的重活)
*   **混合布尔运算**: **`manifold3d`** (绝对可靠)
*   **物理寻优与支撑**: `scipy.optimize` + `trimesh.ray_pyembree`

这份技术预研极大提升了我们在这个项目的胜率。传统软件之所以无法被替代，是因为它们是一个重型的单体（Monolith）。而我们的 LangGraph 架构，由于其可插拔的特性，使得我们能够像拼乐高一样，把 `Hunyuan`、`MeshLib`、`Manifold3D` 这些散落在世界各地的单项冠军组合在一起，形成**超越任何单一传统软件能力边界**的超级管线。
